# StarAI Practice Client Roadmap

## 0. 다시 시작 기준

- 기존 구현은 제거하고, 새 Core 정책과 테스트를 기준으로 재구축한다.
- 원본 설치 폴더는 읽기 전용으로만 다룬다.
- 매 단계마다 단위 테스트와 smoke를 통과시킨다.
- 구현 방식이 여러 개인 기능은 `docs/TECH_DECISIONS.md`에 후보/장단점/선택 이유/검증 방법을 기록한 뒤 진행한다.

## 1. 카탈로그와 선택 모델

- SCHNAIL `bots.dat`와 `maps.dat`를 읽어 봇/맵/종족/BWAPI 버전/ELO/맵 호환성을 구성한다.
- 봇을 먼저 고르면 가능한 맵만, 맵을 먼저 고르면 가능한 봇만 남긴다.
- 사용자 추가 맵은 별도 사용자 맵 폴더로 받되, 원본 SCHNAIL 맵 폴더는 수정하지 않는다. 현재 `.scm/.scx` 읽기와 런타임 복사를 지원한다.

## 2. 런타임 준비

- 사람 런타임 `C:\starai\SC116AI`와 AI 런타임 `C:\starai\SC116AI_ai`를 분리한다.
- 핫키는 SCHNAIL CSV에서 가져와 편집하고, SFmpq 방식으로 작업용 런타임 MPQ에만 반영한다.
- `bwapi.ini`, `wmode.ini`, replay 경로, sound 설정을 역할별로 생성한다.

## 3. 실행 오케스트레이션

- 사람 클라이언트가 방을 만들고 AI 클라이언트가 참가하는 순서를 안정화한다.
- AI 클라이언트는 창모드, 음소거, APMAlert OFF를 기본값으로 둔다.
- ChaosLauncher 중복 실행과 프로세스 정리 범위를 `C:\starai` 아래 로컬 런타임으로 제한한다.
- 실제 실행 smoke와 스크린샷으로 QHD 전체 창모드/AI 관찰 창을 검증한다.

## 4. 화면/마우스

- 독점 전체화면을 쓰지 않고 W-MODE 테두리 없는 전체 창모드를 구현한다.
- QHD/Windows 11 환경에서 해상도 강제 변경과 다른 창 재배치를 일으키지 않는지 검증한다.
- 마우스 감도는 창 크기/스케일링과 W-MODE 설정을 먼저 조사하고, 게임 종료 시 Windows/Logitech 설정으로 자연 복귀해야 한다.

## 5. 연습 기능 확장

- 래더 모드: 현재 실행 세션/맵/종족/APM 기록을 저장한다. 승패 자동 판정과 점수 계산은 후속이다.
- 봇 빌드 선택은 봇별 설정 파일 구조를 확인한 뒤 지원 가능한 봇부터 시작한다.
- 봇 난이도는 SCHNAIL ELO와 자체 전적을 함께 쓰고, 가능하면 한국 서버 래더 MMR/등급 기준으로 병행 표기한다. 환산표는 근거를 확인하고 설정 가능하게 둔다.
- 리플레이 자동 저장 경로와 매치 히스토리를 연결한다.
- 인게임 타이머/APM은 APMAlert 대신 외부 오버레이로 표시한다.

## 6. Remastered 가능성 조사

- Remastered 직접 실행은 별도 조사 항목으로 둔다.
- 1.16.1/BWAPI 안정 런처가 먼저 동작한 뒤, 호환성과 법적/기술적 제약을 분리해 검토한다.
